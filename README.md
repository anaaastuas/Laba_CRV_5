Лабораторная работа №5: Реализация кастомного мьютекса

Описание работы
1. Класс CustomMutex
Реализован на основе std::atomic<bool> и метода compare_exchange_weak() для обеспечения взаимного исключения.

Ключевые особенности:

Метод lock():
Использует атомарную операцию compare_exchange_weak() для безопасного захвата мьютекса. Если мьютекс уже заблокирован, поток вызывает std::this_thread::yield(), что снижает нагрузку на процессор.

Метод unlock():
Сбрасывает флаг locked в false с использованием std::memory_order_release для обеспечения синхронизации памяти.

Преимущества реализации:

Взаимное исключение: Гарантирует, что только один поток может находиться в критической секции.

Отсутствие активного ожидания: Использует yield() вместо бесконечного цикла, что снижает нагрузку на CPU.

Кросс-платформенность: Использует только стандартные средства C++ без платформенно-зависимых вызовов.

Программа состоит из следующих компонентов:
